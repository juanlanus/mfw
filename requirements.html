non-functional

logging

All interactions are logged. The log entries must have enough information in order to identify the author, and their effect (like, for example, to be able to revert it by hand). Also, the log database must be read-only to averybody.

This is a semantical security feature. THe client company would allow its employees, especially the admins, to be able to edit data that if incorectly manipulated might damage other employees careers.

This includes all the administration and evaluations data, that is, pretty much everything.

Those exerting such power must know that they are being logged, that is, that if they make an error the information to fix it will be available, and that if they attempt something dishonest they will be leaving traces clear behind them.

Interactive data must include:

  • user
  • timestamp
  • workstation identification
  • URI
  • raw request data (with any passwords obscured)
