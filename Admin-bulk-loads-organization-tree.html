Admin bulk-loads organization tree

 

Admin runs a procedure (like a script or a special program) that reads the tree data and stores it in the database.

This is a one-time task, needed to start the use of the application.

If the data volume is small then it can be done using the regular CRUD interactions.  

This task involves creating the users and their "reports to" relationships.

All the users are initially assigned to a not-assigned group. After, they are moved to the group of the corresponding manager.
